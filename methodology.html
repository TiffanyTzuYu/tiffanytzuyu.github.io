My Work Methodology

My approach aligns with AGILE practices.
Translating business needs into clear, actionable requirements and delivering them through continuous stakeholder collaboration.

Work Mind Map

I like to map out my thoughts before starting work—it helps clear my mind, clarify the steps needed, and improve planning. Working smart helps me achieve more with less effort.

Methodology Breakdown

Phase 1

Stakeholder Engagement & Requirements Gathering

  • Conduct stakeholder interviews to surface pain points
  • Define project scope, objectives, and success criteria
  • Map out the current-state & to-be processes
  • Identify key data sources, systems, and integration points
  • Elicit functional and non-functional requirements
  • Validate requirements with sign-off from business owners
Phase 2

Data Exploration & Analysis

  • Understand data sources and access methods (e.g., databases, APIs, or manual Excel inputs)
  • Identify data gaps, inconsistencies, and transformation needs; digitize any manual data sources where possible
  • Design the data model based on user requirements, including SQL queries and API integrations
  • Understand table structures, define entity relationships, and build the data model
Phase 3

Solution Design & Prototyping

  • Design data pipelines and ETL/ELT transformation logic
  • Build Power BI prototype dashboards for early stakeholder feedback
  • Define KPIs, metrics, and visualisation standards
  • Collaborate with developers to translate requirements into system specs
Phase 5

Testing & Validation

  • Write and execute UAT test cases aligned to acceptance criteria
  • Validate data accuracy by reconciling reports against source systems
  • Fix if any data issues, discrepancies, and measurement inconsistencies.
  • Facilitate sign-off with business stakeholders before go-live
Phase 6

Delivery, Monitoring & Iteration

  • Create end-user documentation and conduct training sessions
  • Monitor dashboard usage and gather post-launch feedback
  • Continuously refine solutions based on evolving business needs

Core Principles

How I approach every engagement — from a small report fix to a full system rollout.

Business First

Understanding business needs by starting with the “why” before defining the “how,” ensuring that technology supports—rather than disrupts—business operations.

Data Integrity

Accurate, consistent, and trustworthy data is non-negotiable. I validate at every stage and document data lineage clearly.

Clear Communication

I translate technical complexity into language stakeholders understand — bridging the gap between IT and business teams.
